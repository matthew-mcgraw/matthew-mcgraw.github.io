Drone Motion Planning

A Python implementation of the SST* (Stable Sparse RRT) motion planning algorithm for navigating a drone through 3D environments — originally built to autonomously follow a snowboarder through dense forestry using dynamically-feasible trajectories.

The system integrates mesh-based collision detection, Monte Carlo propagation, and witness-region pruning to produce sparse yet near-optimal trajectory trees under realistic vehicle dynamics.

Project Subproblems

Subproblem 1 — No Dynamics

Baseline

A kinematic-only planner that navigates the drone through the flat-slope forestry scene. Collision checking is performed by sampling sphere proxies along candidate paths against the environment STL mesh. Serves as the foundation for understanding SST tree growth and pruning.

Subproblem 1 — With Dynamics

Full Model

Adds a full nonlinear drone dynamic model. Trajectories are integrated using scipy.odeint over randomly sampled propagation times. Pitch and velocity bounds are enforced at every integration step before a new vertex is accepted into the active set.

SST* Core Algorithm

Algorithm

Implements the Stable Sparse Tree planner with best-first node selection, witness regions for pruning dominated vertices, and Monte Carlo random control propagation — yielding sparse, near-asymptotically-optimal path trees.

MotionPlanningAlgorithms/SST.py δ_s = 1.5

How SST* Works

01

Sample State

Uniformly sample a random state inside the bounding mesh volume. With 5% probability, bias toward the goal.

02

Best-First Select

Find the lowest-cost active vertex within δ_bn of the sample. If none exist, extend toward the closest node.

03

Monte Carlo Propagate

Apply random control inputs (yaw rate, pitch rate, linear acc) integrated over a random time t ∈ [0, T_prop].

04

Prune & Witness

Check if the new node beats its witness region representative. If so, demote the old node to the inactive set.

Drone Dynamics Model

Trajectories are forward-integrated using a 6-DOF nonlinear drone model with coupled pitch, yaw, and velocity states. The equations of motion are solved with scipy.odeint at each propagation step. 

# Drone equations of motion (odeint integration)


def dXdt(X, t):
x, y, z, yaw, theta, v = X
return [
v * cos(yaw) * cos(theta),   # ẋ
v * sin(yaw) * cos(theta),   # ẏ
v * sin(theta),              # ż
omega,                       # ψ̇  (yaw rate control)
alpha,                       # θ̇  (pitch rate control)
acc                          # v̇  (linear acceleration control)
]

# Propagate over random time t ∈ [0, T_prop=8s], 10 integration steps
t = np.linspace(0, t_sampled, 10)
sol = odeint(dXdt, x_0, t)

Mesh Collision Detection

Environment Mesh

STL

The forestry scene is loaded as a full STL mesh (FlatSlopes_WithTrees.stl). A persistent trimesh.CollisionManager is initialized once at startup and reused across all 10,000 iterations for efficiency.

Sphere Proxy Checking

Per Trajectory

At each integration step along a candidate trajectory, an icosphere of radius = drone max dimension / 2 is placed at that position and tested for internal collision against the environment geometry.

Bounding Volume

State Validity

A separate bounding volume mesh (FlatSlopes_BoundingVolume.stl) constrains all sampled states. Only states inside this volume are valid — enforced with trimesh.sample.volume_mesh.

About This Project

Originally developed as the final project for Advanced Motion Planning at CU Boulder (Dec 2022). The goal was to plan collision-free, dynamically-feasible drone trajectories through a dense forestry environment to follow a snowboarder. The codebase was later cleaned up and expanded with a modular SST class, improved pruning, and a SceneBuilder utility for exporting trajectory point clouds as STL files.
